Author: kejiqing

**目的：** 把当天反复踩坑收敛成一张表，避免再绕圈改插件 / compose / Machine settings。  
**操作手册仍以** [EXTENSION-STABLE-DEPLOY.md](./EXTENSION-STABLE-DEPLOY.md) **为准**；本文是「为什么颠」的证据链。

## 1. 成功标准（别争别的）

浏览器打开 `proj_N/home`，Chat 发：

```
@claw ping
```

应看到 Claw 回复（如 `pong`），**不是**下面任一报错：

- `No activated agent with id "claw.claw"`
- `Language model unavailable`
- `claw.projId not set`
- `agent WebSocket error (no gateway response …)`

再测：

```
@claw 看看claude.md 里面有什么
```

应出现 agent 工具调用（Glob/Read）+ 对 `CLAUDE.md` 的中文说明。

## 2. 症状 → 根因 → 修复（证据链）

| 症状 | 根因 | 怎么证实 | 修复 |
|------|------|----------|------|
| `No activated agent` | ① `chat.agent.enabled: false` ② 只 unzip 未 `--install-extension` ③ **JSDoc 写 `proj_*/home` 触发 `*/` 提前结束注释** → `extension.js` 语法错误，扩展从未 activate | Chrome Console: `Activating extension 'claw.claw-vscode' failed: Unexpected token '.'`；`node --check extension.js` 失败 | 修 JSDoc（注释里写 `proj_N/home`，不要 `proj_*`）；`ovs-claw-restart.sh` |
| `Language model unavailable` | 删掉 **stub LM**（`registerLanguageModelChatProvider` + `chatProvider` proposal） | participant 能 @ 到，但 Chat UI 报 LM 不可用 | **必须保留** stub LM；与 JSDoc 修正是两件事，不能二选一 |
| `claw.projId not set` | 工作区不是 `proj_N/home`（打开了 `/home/workspace` 根） | OVS 标题/Explorer 显示 `workspace`，并列 `proj_1`/`ds_*` | 走 Playground `18765/ovs?projId=N` → 302 到 `?folder=.../proj_N/home`；**已删** compose `--default-folder=/home/workspace` |
| Playground 进 OVS 仍是整盘 workspace | 经 `:18765` **代理 HTML** 时 OVS 拿不到 `folder=` | URL 无 `folder=`；Explorer 根是 `/home/workspace` | `server.py` 对 `/ovs` **302 到 :13000/?folder=proj_N/home**（已落地） |
| `agent WebSocket error (no gateway response)` | ① OVS 容器 **不在 `claw_default` 网络**，`gateway-rs` DNS `ENOTFOUND` ② 浏览器误连 `ws://:13000/ovs/agent/ws`（OVS 无此路由）③ pool `ovs-1` ttyd 僵死，registry 有记录但 WS 不通 | `podman exec OVS node -e "require('dns').lookup('gateway-rs'…)"` → ENOTFOUND；`podman inspect` 网络 `stack_default` vs `claw_default`；浏览器直连 `ws://127.0.0.1:8088/.../agent/ws` 可通而 EH 路径不通 | `podman network connect claw_default claw-openvscode-server`；**禁止**裸 `podman-compose up` 不带 `COMPOSE_PROJECT_NAME=claw`；扩展 **0.2.6+** 浏览器走 `claw.gatewayPublicHost`（默认 `127.0.0.1:8088`）；僵死时 `POST .../terminal/stop` + `terminal/start` 或 `gateway.sh pool-reset` |

## 3. 三层契约（搞混就会返工）

```
浏览器 URL          → 必须 proj_N/home（Playground 302 或手带 ?folder=）
projId              → 只认 Gateway 写入 proj_N/home/.vscode/settings.json
agent WebSocket     → Remote EH: ws://gateway-rs:8080/...（要同 Docker 网络）
                      浏览器页在 :13000: ws://127.0.0.1:8088/...（claw.gatewayPublicHost）
                      浏览器页在 :18765: ws://:18765/ovs/agent/ws（Playground 代理）
```

**插件本身（participant + stub LM）在以上对齐后无需反复改。**

## 5. 禁止事项（当天浪费 token 的来源）

| 不要做 | 为什么 |
|--------|--------|
| compose / Machine / env 写死 `claw.projId` | 多项目必乱；违反 Gateway 契约 |
| 只改插件猜 folder / projId | 根因在 URL + Gateway settings，不在 participant 逻辑 |
| 裸 `deploy/stack` 下 `podman-compose up` 重建 OVS | 易落到 `stack_default`，断 `gateway-rs` |
| 删掉 stub LM「简化插件」 | 必现 `Language model unavailable` |
| JSDoc 里写 `proj_*/home` | `*/` 结束注释 → 扩展语法错误 |
| 用 `extensions.user.cache` 当唯一安装信号 | OVS 1.109.5 上常无此文件；改查 `list-extensions` + `node --check` |
| E2E 过了就宣称浏览器好了 | 必须浏览器发 `@claw` 才算 |

## 6. 扩展 0.2.6 要点（`extensions/claw-vscode`）

- `registerLanguageModelChatProvider("claw.lm", …)` + `enabledApiProposals: ["defaultChatParticipant","chatProvider"]`
- `createChatParticipant("claw.claw", …)` → gateway agent WS
- `agentWsUrl()`：`:18765` → Playground 代理；`:13000` → `claw.gatewayPublicHost`（`127.0.0.1:8088`）；Remote EH → `claw.gatewayHost`（`gateway-rs:8080`）
- Machine settings（`openvscode-settings.json`）新增：`claw.gatewayPublicHost`、`claw.playgroundPort`
