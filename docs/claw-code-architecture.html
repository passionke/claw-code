claw-code 网关与可观测性改造架构

覆盖异步任务 API、MCP 路由与缓存、Trace 链路、模型边界日志和外部系统调用链路

改造重点 1：异步化调用链

新增 /v1/solve_async 与 /v1/tasks/{taskId}，网关快速返回任务 ID，长时分析通过轮询获取状态与结果，避免同步接口硬超时。

改造重点 2：MCP 稳定性

Runtime 增加 mcp discovery 缓存与工具路由预热；stdio 初始化支持 framing fallback，降低 initialize 握手失败与 unknown tool 误差。

改造重点 3：端到端可观测

Gateway + Runtime 统一 trace 事件流（decision/tool/data/assertion），并暴露 trace API / viewer；补充 child_log_full 用于故障后的整段日志复盘。
